# Архитектура СУРВ (учёт времени + биометрия)

## Клиент: биометрия по лицу

Целевой конвейер распознавания (устойчивость к смене фона за спиной):

1. **BlazeFace** — детектор лица на полном кадре с камеры: находит область лица (bounding box / ключевые точки).
2. **Canvas** — по координатам из BlazeFace вырезается **только регион лица** (crop). В эмбеддинг попадает преимущественно лицо, а не комната, мебель и т.п. Благодаря этому при смене фона позади пользователя признаковое описание остаётся согласованным с эталоном «его лицо», а не «он в этой комнате».
3. **MobileNet v2** (TensorFlow.js, слой активаций `conv_preds`) — по **обрезанному** изображению с canvas строится вектор признаков текущего кадра. Этот вектор дальше используется в KNN / сравнении с эталоном сотрудника.

Сотрудники, смены, логи и KNN хранятся в **MySQL** (`db.js`, пакет `mysql2`). Клиент обращается только к REST API (`public/surv-api.js`); IndexedDB (`local-db.js`) больше не используется — данные нельзя стереть через F12.

Подключение по умолчанию: пользователь `test`, пароль `1234`, БД `surv` (см. `.env.example`, переменные `MYSQL_*`). При старте сервера таблицы создаются автоматически.

### Реализация в коде

Пайплайн лица: `public/face-pipeline.js` (`SurvFacePipeline`) — BlazeFace → canvas → MobileNet v2 → KNN; результат сохраняется через `POST /api/knn-dataset`.
